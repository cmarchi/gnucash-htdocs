The GnuCash Documentation Set

The GnuCash documentation is managed as a BOOK, composed of a set of DocBook "ARTICLES."

The main document is gnucash.sgml, which really contains only the "sequencing" information on what files are to be consulted to make up the body of the document, each file representing one of the ARTICLEs.

Note that filenames for the HTML output files are generated in three ways:

  • The ID tag for each ARTICLE defines a filename;

  • The ID tag for each SECT1 defines a filename, unless it is the first SECT1 in an ARTICLE;

  • If you create a new ARTICLE or SECT1 without an ID tag, filenames will be generated with names like t1234.html, x892.html, and such. They will look quite random.

    This is a bad thing; make sure you assign an ID tag to the ARTICLE or SECT1.

Styling is controlled by the DSSSL document, mysheet.dsl, using the conventions documented in the Modular DocBook Stylesheets

Components that need to be installed in order for the command jade -ioutput.html -d mysheet.dsl -t sgml gnucash.sgml to work include:

  • Jade or DSSSL/OpenJade

  • The DocBook 3.1 DTD, available from DocBook.org

  • Norman Walsh's Modular DocBook Stylesheets
